# Community Hero: Hyperlocal Problem Solver

Community Hero is a production-grade, highly polished, full-stack civic action platform designed for citizens to identify, report, validate, track, and resolve local municipal infrastructure and environmental issues (such as potholes, water leaks, broken streetlights, and waste pile-ups).

By combining interactive mapping, real-time Firestore database synchronization, gamified civic participation, and server-side **Gemini AI Vision** automated validation, the platform transforms passive complaining into coordinated, transparent civic resolution.

---

## 🌟 Core Features

### 1. AI-Powered Report Synthesis (The "Zero-Effort Reporter")
*   **Intelligent Image Analysis**: Drag-and-drop an image or take a photo of an issue. The client sends it securely to our Express backend.
*   **Structured Gemini AI Categorization**: The backend calls `gemini-3.5-flash` using the official `@google/genai` SDK to analyze the photo and returns:
    *   **Standardized Category**: Automatically parsed into `Pothole`, `Water Leakage`, `Broken Streetlight`, `Waste Management`, or `Infrastructure`.
    *   **Severity Rating**: Ranked `Low`, `Medium`, `High`, or `Critical`.
    *   **Professional Summary**: A concise description of the incident.
    *   **Safety & Actionable Advice**: Real-time cautionary tips generated by Gemini (e.g., "Avoid walking near the base of this post due to potential electrical exposure").
*   **Smart Geolocation**: Autocompletes precise device coordinates (`lat`, `lng`) or allows searching for an address with a beautiful Google Maps search bar.

### 2. Interactive Civic Map & Feed (The "Civic Radar")
*   **Advanced Google Maps Integration**: Powered by `@vis.gl/react-google-maps` using responsive markers, info boxes, and custom SVG category icons.
*   **Fallback Interactive Grid**: Includes a fully interactive SVG Grid Map canvas when a Google Maps API Key is not configured, ensuring 100% visual fidelity and testing usability in any sandbox.
*   **Bento-style Action Feed**: Allows filtering by status (`Reported`, `Validated`, `In-Progress`, `Resolved`), category, and severity with fluid search controls.

### 3. Verification & Civic Resolution Engine (The "Consensus Guard")
*   **Crowdsourced Upvotes**: Neighbors can verify a reported issue to prevent spam.
*   **Status Progression**:
    *   `Reported` -> `Validated` (Automatically upgrades at +3 community upvotes).
    *   `In-Progress` (Flagged when volunteers or workers claim it).
    *   `Resolved` (Closed when a user uploads an "After Photo", featuring a Before/After visual comparison slider).

### 4. Live Civic Analytics & Gemini Trend Synthesis
*   **Statistical Charts**: Live visualization of report ratios by category, status timelines, and severity heatmaps using `recharts`.
*   **Weekly Trend Report**: Runs an on-demand regional synthesis model asking Gemini to parse local coordinates and issue distribution, delivering proactive infrastructure advice (e.g., advising main water valve repairs near clustered leakage locations).

### 5. Gamification & Citizen Profile
*   **Points Ledger**: Citizens earn points for helping their city (+10 for reporting, +2 for verifying, +5 for helpful comments, +50 for resolving).
*   **Civic Badges**: Collectable milestones such as `Pothole Ranger`, `Eco Warrior`, `Beacon of Light`, and `Civic Sentinel`.
*   **Real-time Leaderboard**: Features a live-sorted, real-time Firestore database subscription ranking community heroes dynamically as points are earned across the entire platform.
*   **Normal Sign-Out & Guest Transition**: Implements clear profile partitioning where signing out safely purges locally cached credentials and gracefully resets back to a fresh "Local Guest" state, preventing any profile leakage across different account sessions.

### 6. Interactive Drag & Drop File Upload
*   **Aesthetic Drop Zone**: Supports drag-and-drop and click-to-upload custom high-resolution images, with interactive scale-up transitions, hover/drag states, and spring animations.
*   **Client-Side Asset Compression**: Automatically compresses uploaded photos client-side to optimize bandwidth and stay within real-time payload thresholds before transmission.
*   **Hardened Cloud Validation Security Rules**: Adjusted Firestore Rules to allow high-capacity base64 compressed strings (up to 1MB) for `imageUrl`, `resolvedImageUrl`, and `avatarUrl` fields while strictly sanitizing other metadata formats.

## 🏗️ Technical Architecture & Security

```
                                 +------------------+
                                 |   Web Browser    |
                                 +--------+---------+
                                          |
                        HTTPS (REST)      |      Client SDK Connect
                     (API Keys Masked)    |      (Optional Auth/Sync)
                                          v
+-----------------------------+  +--------+---------+  +-------------------------+
|    Express Backend (Port 3000)  |  | Firebase Auth    |  | Firebase Firestore DB   |
+--------------+--------------+  +------------------+  +-------------------------+
               |
               |  @google/genai Client
               v
+--------------+--------------+
|     Gemini 3.5 Flash Model  |
+-----------------------------+
```

### 1. Robust Dual-Mode Architecture
To guarantee reliability, the platform operates a **dual-mode persistence and authentication engine**:
*   **Cloud Synchronized Mode**: Uses Firebase Authentication and Firebase Firestore real-time collections if configured.
*   **Hyperlocal Offline Mode**: Gracefully falls back to browser-persistent `localStorage` if Firebase is unprovisioned, preventing crashes and keeping the application fully functional.

### 2. Secret Key Isolation
*   **Server-Side Secret API Proxy**: The `GEMINI_API_KEY` is kept strictly server-side inside `server.ts`. No frontend component ever makes direct calls to Google Generative AI endpoints.
*   **Controlled Client Exposure**: Only client-safe variables (prefixed with `VITE_`) are accessed on the frontend.

## 💾 Firestore Collection Schemas

*   **`users/{userId}`**: Profiles containing point balances, badge arrays, and timestamps.
*   **`reports/{reportId}`**: Main report tracking including GPS coordinates, categorization, severity, images (Before/After), and upvote arrays.
*   **`reports/{reportId}/comments/{commentId}`**: Crowdsourced comments on active tickets.

## 🛠️ Build & Start Instructions

### Development Runtime
```bash
npm run dev
```
Runs the Express application utilizing Vite middleware on port 3000.

### Production Compiling & Execution
```bash
npm run build
npm start
```
1. Builds the static client assets in `dist/`.
2. Bundles the TypeScript `server.ts` into a self-contained CommonJS file at `dist/server.cjs` using `esbuild` to bypass runtime import constraints.
3. Serves the production build seamlessly via standard Node.js execution.
